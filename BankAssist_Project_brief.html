What Changed in v2.0
Persona pivot: The project has moved from an internal bank investment advisory officer tool to a customer-facing chatbot. Customers authenticate themselves, ask for account statements over a date range, query individual transactions, and download a CSV — all through a Streamlit chat UI. The three deep learning components (BERT intent classifier, T5 Text-to-SQL, FAISS RAG) remain unchanged and continue to satisfy the rubric. The main additions are an authentication layer (SQLite customer lookup), a statement generation engine (pandas date-range filter → CSV), and the Streamlit front end.
System Architecture
Input
Streamlit Chat UI
st.chat_input, st.chat_message, st.session_state
Sidebar: account number / mobile / customer ID login form
↓
NLP
BERT Intent Classifier
Fine-tuned bert-base-uncased. 5 classes: GENERATE_STATEMENT, QUERY_TRANSACTIONS, AUTHENTICATE, HELP, DOWNLOAD_CSV
NER / Date Extractor
spaCy + regex extracts date ranges ("last 3 months", "Jan to March"), account references, and transaction keywords from free-form text
↓
Router
Intent Router
Routes each turn to the correct handler based on BERT output. Unauthenticated requests are redirected to AUTH handler regardless of intent.
↓
Data
Transaction DB Handler
synthetic_transactions.db (SQLite). pandas date-range filter. Returns DataFrame for statement and CSV export via st.download_button
T5 Text-to-SQL
Fine-tuned t5-small on Spider + synthetic finance pairs. Converts NL transaction queries to SQL. Executed against synthetic_transactions.db
FAISS RAG (Help Branch)
all-MiniLM-L6-v2 embeddings. Retrieves FAQ / policy / product docs for HELP intent. Returns top-k passage as bot reply
↓
Output
Chat Response
Structured text reply via st.chat_message("assistant")
CSV Download
st.download_button delivers statement as account_statement.csv
matplotlib Chart
Optional: st.pyplot() renders spending trend or transaction bar chart when user requests "show me a chart"
↓
Training
BERT Fine-tuning
HuggingFace Trainer API. intent_classification_v2.csv (600 rows, 5 classes). Metric: weighted F1.
T5 Fine-tuning
Seq2seq training. text_to_sql_transactions.csv (200+ pairs). Metric: exact-match SQL accuracy.
RAG Index Build
faiss-cpu index over FAQ / policy documents. Eval: retrieval precision@3 on held-out questions.

Deep Learning Models
1. BERT Intent Classifier
Base: bert-base-uncased (HuggingFace)
Task: 5-class intent classification
Training: intent_classification_v2.csv, 80/20 split
Epochs: 3  |  LR: 2e-5  |  Batch: 16
Eval: Weighted F1, confusion matrix
Framework: HuggingFace Trainer API
2. T5 Text-to-SQL
Base: t5-small (HuggingFace)
Task: Seq2seq NL → SQL generation
Training: Spider finance + synthetic pairs
Epochs: 5  |  LR: 3e-4  |  Batch: 8
Eval: Exact-match SQL accuracy
Execution: sqlite3 against transactions.db
3. FAISS RAG (Help)
Embeddings: all-MiniLM-L6-v2
Index: faiss-cpu (cosine similarity)
Corpus: 30+ FAQ / policy documents
Retrieval: top-3 chunks per query
Eval: Retrieval precision@3
Triggered by: HELP intent from BERT
Stretch Goal — LSTM Autoencoder (Anomaly Detection): If time permits after Day 7, Pair B can add an LSTM Autoencoder trained on customer transaction sequences to flag statistical anomalies (unusually large single transactions, unusual merchant categories). Adds a third DL architecture to the project and strengthens the rubric score under "model depth and variety."
