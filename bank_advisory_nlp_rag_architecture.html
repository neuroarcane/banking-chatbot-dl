Datasets — All Pre-Generated, No Live API Calls Required
Note: All datasets are already generated and saved in the /data folder. There are no live API dependencies — Yahoo Finance and FRED have been replaced by a pre-built synthetic ETF price dataset with realistic geometric Brownian motion parameters. The schema is identical, so the SQLite database works as a drop-in replacement.
synthetic_client_profiles.csv
Pre-generated
5,000 rows · 11 features
Age, income bracket, risk tolerance (1–5), horizon, ESG flag, equity %, bond %, invest amount → portfolio label. 5% noise added. Used to train the portfolio classifier.
etf_returns.db
Pre-generated
SQLite · 5 tables · 776 KB
13,145 daily price rows for SPY, AGG, ESGV, VNQ, GLD (synthetic GBM). Annual returns, blended portfolio returns, CAGR summaries. Powers the Text-to-SQL branch.
intent_classification.csv
Pre-generated
500 rows · 4 balanced classes
Labelled financial advisory questions: RECOMMENDATION, DATA_QUERY, QUALITATIVE, CLARIFICATION (125 each). Used to fine-tune the BERT intent classifier.
text_to_sql_finance.csv
Pre-generated
198 NL→SQL pairs
Natural language questions mapped to executable SQL targeting all 5 tables in etf_returns.db. Covers aggregations, JOINs, rankings, time-range queries. Used to fine-tune T5.
Spider (finance subset)
HuggingFace Hub
~2,000 pairs (filtered)
Finance-adjacent schemas from Yale NLP Spider dataset, loaded via datasets library. Supplements text_to_sql_finance.csv for T5 fine-tuning coverage of JOIN queries.
Financial PhraseBank
HuggingFace Hub
4,845 sentences
Finance sentences labelled positive / negative / neutral. Optional supplement to the intent classifier training set for richer domain vocabulary coverage.

10-Day Sprint Timeline — 4–6 Hours Per Day
1Kickoff
Kickoff
All members. Clone repo, set up venv + requirements.txt, explore all 4 datasets in Jupyter. PM scaffolds folder structure with Claude Code. GitHub Project board created. Branch naming agreed.
2Parallel Build
Parallel Build
Pair A: label 200 intent samples; BERT tokeniser check.  Pair B: Spider schema exploration; 10 manual SQL queries against etf_returns.db.  Solo (Portfolio): full EDA on client profiles — class balance, pairplot, correlation matrix.  PM: chunk + embed 20 RAG docs into FAISS.
3First Train
First Training
Pair A: first BERT run (3 epochs); log baseline F1.  Pair B: first T5 run; test Exact Match on holdout.  Solo: MLP v1; confusion matrix; start hyperparameter log.  PM: FAISS retrieval end-to-end; router stub by intent class.
4Iterate
Iterate + Improve
All models get a second training pass. Pair A: full 500-sample BERT run, adjust learning rate. Pair B: augment with Spider finance subset. Solo: LSTM alternative vs MLP comparison. PM: output formatter (text + chart selector). Use Claude Code to auto-generate evaluation scripts.
5Integrate
Integration Day
Target: full pipeline runs without errors on 10 curated test queries. PM merges all branches. Each member fixes their own blockers. End-to-end smoke test covering all 4 intent classes. Document every failure mode.
6Polish
Polish + Edge Cases
Pair A: ambiguous intent handling, confidence threshold. Pair B: SQL error recovery (re-prompt on syntax error), expand chart types. Solo: finalise EDA notebooks; write Dataset + Model report sections. PM: 30-query stress test; fix router edge cases.
7Demo Build
Demo Build
Full 20+ query demo walkthrough. Fix top 5 failure cases. Solo completes report draft (all sections except Results). Each pair writes their model comparison narrative. PM builds slide deck skeleton. Cursor used to clean up all code (docstrings, type hints, remove debug prints).
8Report
Report + Rehearsal
Report writing sprint. PM compiles into final template. First full 15-min rehearsal — time each section. All notebooks verified to run clean top-to-bottom. Contribution table filled in precisely. No new features after this point.
9Present
PRESENTATION
Live class presentation — 15 min + 5 min Q&A. Demo: 3 live queries — one RECOMMENDATION (client profile → portfolio), one DATA_QUERY (SQL → line chart), one QUALITATIVE (RAG response). PM presents architecture; each member presents their model results. Do not change any code after this session.
10Submit
Final Submission
Final polish only — no new features. Run all notebooks top-to-bottom one last time. Verify all paths are relative. Solo compiles zip: /data + /notebooks + /models + /rag + /output + /report + /slides. PM checks against rubric. Submit on Blackboard by EOD.
